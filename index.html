DSA Practice

Why this project?

To showcase my progress in leetcode.

Why the tech stack?

I chose a full-stack framework like Next.js for its built-in routing, API support, and server-side rendering. Pairing it with TypeScript helped catch errors early and kept the codebase maintainable.

Although I had no prior experience with Next.js and only a basic understanding of TypeScript, building this project gave me hands-on experience. I'm still learning, but I now have a strong grasp of how both tools work together and individually.

Challenge:

Working with @monaco-editor and vm2. The tests were generated by chatgpt and gathered from leetcode, but problems have different parameters and I had to figure out how to spread the parameters correctly. In short, I overcomplicated it.

What I would add:

As of right now, I have only completed easy leetcode problems. In the future, I be doing medium and hard problems. This means the tests will need to be categorized accordingly.

Another thing I'd like to add is the choice to use a difference language. Languages I want to answer these problems in are Python and C#.
